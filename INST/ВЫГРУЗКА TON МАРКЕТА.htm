TON Storage без сервера: самый простой рабочий вариант

Идея: хостим чисто статический билд в TON Storage и привязываем его к lfgsyndicate.ton. Сервер не нужен. TonConnect/платежи оставляем на HTTPS (Vercel) для 100% совместимости с кошельками.

Что учесть

  • TON Storage — это “IPFS‑подобный” хостинг: нужен один HEX‑идентификатор контента. За хранение платится небольшой фи (часто копейки TON через загрузчики).
  • Сайт должен быть полностью статическим. У нас так и есть (Vite build → dist).
  • Манифест TonConnect лучше оставить на Vercel (HTTPS), чтобы все кошельки его гарантированно тянули.

Шаги

  1. Соберите билд (он уже собирается на Vercel)
  • Локально: npm run build в папке TON_LFG → получится dist/
  • Или скачайте dist из последнего деплоя Vercel (Artifacts/Download)
  1. Загрузите dist/ в TON Storage
  • Используйте любой надежный загрузчик в экосистеме TON (бот/веб‑сервис). Процедура обычно:
    • Заархивируйте dist или загрузите как папку
    • Оплатите микрофи (если требуется)
    • Получите “TON Storage HEX” (длинная hex‑строка контента)
  • Важно: убедитесь, что в dist лежат:
    • index.html
    • папка /assets/*
    • favicon.*
    • tonconnect-manifest.json тоже скопируется (Vite кладёт всё из public в корень dist)
  1. Привяжите HEX к домену
  • dns.ton.org → ваш домен lfgsyndicate.ton
    • Wallet address: UQC1WXkJ_7t7sGu6ZTZ9BGoR6YAwtPoKoUf7KZtrgOQ3w7km → Save
    • TON Site (ADNL): оставьте пустым (нам не нужен сервер/прокси)
    • TON Storage: вставьте полученный HEX → Save
    • Hosting in TON Storage: не нужен (вы уже задали HEX в поле TON Storage)
  1. Проверка
  • В браузере/Telegram (с TON Proxy) откройте: https://lfgsyndicate.ton
  • DevTools → Network: ассеты грузятся из /assets/*.js|css, типы application/javascript/text/css
  1. TonConnect и платежи
  • В коде уже указано:
    • manifestUrl: https://lfg-ton-marketplace.vercel.app/tonconnect-manifest.json — так и оставляем (самый надёжный вариант)
    • получатель: UQC1WXkJ_7t7sGu6ZTZ9BGoR6YAwtPoKoUf7KZtrgOQ3w7km
  • Платёжные ссылки:
    • ton://transfer/lfgsyndicate.ton?amount=10
  1. Обновление версии
  • При новом релизе: снова загрузите свежий dist/ → получите новый HEX → обновите поле “TON Storage” в DNS → сразу заработает новая версия по lfgsyndicate.ton.

Почему так проще и эффективно

  • Не нужен сервер/прокси/настройки UDP/ADNL.
  • Вся логика TonConnect остаётся на обычном HTTPS (Vercel) — это максимально совместимо с кошельками и Telegram.
  • Сайт открывается по .ton внутри экосистемы TON, статично и быстро.

 

 

 

 

 

 

Короткий чек‑лист перед выгрузкой в TON Storage

  1. Билд
    • В папке TON_LFG:
    •  

o    npm ci

o    npm run build

 

    • Проверить, что в dist/ есть: index.html, /assets/*, favicon.*, tonconnect-manifest.json.
  1. Быстрая упаковка
    • Windows PowerShell:
    •  

o    Compress-Archive -Path dist\* -DestinationPath dist.zip -Force

    • macOS/Linux:

o    cd dist && zip -r ../dist.zip . && cd ..

o     

  1. Публикация в TON Storage
    • Загрузить dist/ или dist.zip через выбранный загрузчик.
    • Получить HEX и вставить в dns.ton.org → lfgsyndicate.ton → TON Storage → Save.
  1. Не менять
    • manifestUrl в коде оставить: https://lfg-ton-marketplace.vercel.app/tonconnect-manifest.json.
    • Адрес получателя: UQC1WXkJ_7t7sGu6ZTZ9BGoR6YAwtPoKoUf7KZtrgOQ3w7km.

Когда будете готовы с правками — пришлите задачи. Подготовлю изменения, соберу билд и доведу до публикации в TON Storage.

    • В списке ваших bag’ов скопируйте длинную hex‑строку (Bag ID).

Как выгрузить папку dist в TON Storage через TON‑Torrent (по вашим скринам)

Что нажимать

  1. В левом нижнем углу нажмите Create
    • Окно “Create Torrent” — это для создания вашего Bag.
    • Окно “Add Torrent” из второго скрина — это для скачивания по Bag ID или .tonbag, нам сейчас не нужно.
  1. В окне Create Torrent:
    • В поле Name введите, например: lfgsyndicate-site.
    • Нажмите Select folder и укажите папку со сборкой:
      • Если билд делаете в подпроекте: c:\Users\user\Desktop\TG\texex-welcome-screen-aero\TON_LFG\dist
      • Если используете корневую сборку: c:\Users\user\Desktop\TG\texex-welcome-screen-aero\dist
    • Убедитесь, что в выбранной папке есть index.html, папка /assets, фавиконки, tonconnect-manifest.json.
    • Галочку Single File НЕ ставьте (мы загружаем папку).
    • Нажмите Continue.
  1. Дождитесь появления записи в списке и статуса завершения:
    • Колонка Progress дойдёт до 100%, статус перейдёт в Seeding.
    • Показатель Peers может быть 0 — это нормально на старте.

Как получить Bag ID (HEX)

  • Кликните правой кнопкой по созданному элементу и выберите Copy Bag ID / Copy ID.
  • Если такого пункта нет: откройте Details/Info у элемента и скопируйте значение поля Bag ID (длинная hex‑строка).

Привязка к домену

  1. Откройте https://dns.ton.org и выберите lfgsyndicate.ton.
  2. Убедитесь, что:
    • Wallet address: UQC1WXkJ_7t7sGu6ZTZ9BGoR6YAwtPoKoUf7KZtrgOQ3w7km → Save.
    • TON Site (ADNL): оставьте пустым.
    • TON Storage: вставьте ваш скопированный Bag ID (HEX) → Save.

Проверка

  • Откройте https://lfgsyndicate.ton в браузере с TON Proxy/в Telegram.
  • Убедитесь, что ассеты грузятся из /assets/* и всё отображается.

Обновления

  • При новом билде снова нажмите Create для новой папки dist → получите новый Bag ID → обновите поле TON Storage в DNS.

Если упрётесь в то, что сайт не открывается или Peers долго 0 — включим “Seed mode” в Settings и (при необходимости) настроим Tunnel в TON‑Torrent. Готов подсказать по месту.
